Day 15 Binary Search — the universal template 4–5 hrs
Slot 1 · 1.5h
learn
The ONE template to memorize:
int lo = 0, hi = n-1;
while (lo <= hi) {'{'}
  int mid = lo + (hi-lo)/2;
  if (condition) return mid;
  else if (arr[mid] < target) lo = mid+1;
  else hi = mid-1;
{'}'}
Why lo + (hi-lo)/2 and not (lo+hi)/2? Overflow prevention. Memorize this. lo <= hi vs lo < hi — when to use each. Lower bound vs upper bound variants. Off-by-one is the #1 binary search bug.

Day 17 Binary Search on Answer (the powerful pattern) 4–5 hrs
Slot 1 · 1h
learn
This pattern changes everything: Instead of searching in an array, you binary search on the ANSWER SPACE. Key insight: "Is it possible to achieve X with Y value?" If you can answer this yes/no check in O(n), then you can binary search for the optimal Y in O(n log n). Applies to: minimum/maximum feasibility problems, allocation problems, capacity problems. The template: define lo=min_possible, hi=max_possible, binary search on feasible(mid).

Day 19 Sliding Window — fixed and variable size 4–5 hrs
Slot 1 · 1h
learn
Two sliding window templates: Fixed size k: sum/max of every window of size k. Use one loop, add right, remove left when window = k. Variable size: expand right pointer always, shrink left when condition violated. Track max/min window satisfying condition. Combined with HashMap for character-based constraints. Key: decide if fixed or variable, then what to track inside the window (sum, count, freq map).

The 3-step recursion framework: (1) Base case — smallest input you can directly answer. (2) What do I do with current element/index.
